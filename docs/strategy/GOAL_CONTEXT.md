# Goal Context

## Mission

`quant-pricer-cpp` exists to become a portfolio-grade, high-performance C++ quantitative finance project: serious option-pricing engines, reproducible numerical validation, benchmark evidence, and public-safe WRDS/OptionMetrics market-data research.

## Long-Term Goal

Make this one of the user's most impressive technical portfolio projects. It should demonstrate C++ systems skill, quantitative finance depth, numerical validation discipline, reproducible artifacts, WRDS data capability, and strong public presentation.

## Near-Term Goal

Run an evidence-backed prestige phase:

1. install the Pro strategy package;
2. refresh and audit current-HEAD evidence;
3. repair public claim/evidence drift;
4. build a larger public-safe WRDS benchmark story;
5. establish the next performance benchmark/optimization frontier.

## Priority Order

1. Prestige/performance with evidence.
2. WRDS/market-data credibility.
3. Numerical correctness where it gates public claims.
4. Public presentation after evidence is true.
5. Minimal process/docs cleanup only where it prevents confusion.

## Non-Goals

- Not a trading system.
- Not a live alpha/performance record.
- Do not commit raw WRDS/OptionMetrics data.
- Do not treat sample WRDS artifacts as live-market proof.
- Do not spend the next phase on broad stale-doc cleanup.
- Do not split ambitious work into tiny conservative tickets.
- Do not strengthen public claims without artifacts, manifests, commands, and review.

## Quality Bar

External-facing numerical, benchmark, WRDS, package, or performance claims require evidence. Evidence means exact commands, current artifacts, manifests/hashes, validation logs, and explicit sample/live boundaries.

## Ambition Posture

Use large coherent Codex tickets. Protect quality through validation, claim audits, and review bundles, not through timid task sizing.

## What Future Sessions Must Not Forget

- User wants maximal prestige/performance and dislikes bureaucracy.
- User has WRDS access and can help gather needed data.
- Some local artifacts/data may have been lost during a server transfer; evidence recovery matters.
- Latest committed metrics snapshot is from 2026-01-25 and must not be assumed current-HEAD.
- T-000 installed AI OS v2 and was approved; do not re-litigate it.
- Archived pre-v2 docs are history, not current truth.
- Live WRDS claims remain gated until explicitly validated and reviewed.
- Heston QE bias remains a known risk.

## Reset Triggers

Return to Pro when public claims change, live WRDS results are promoted, reproduction contradicts existing metrics, Heston/BS narrative changes, benchmark claims become external-facing, raw data risk appears, or after 3-5 substantial Codex review bundles.
